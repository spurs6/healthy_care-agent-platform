# OpenEvidence 临床证据助手 - 前后端启动指南

## 一、环境要求

### 1.1 系统环境
- 操作系统：Windows 10/11
- Python：3.10+
- Node.js：18+
- npm：9+

### 1.2 Python依赖安装
```powershell
# 在项目根目录执行
pip install fastapi uvicorn sse-starlette python-dotenv python-multipart
pip install sentence-transformers chromadb
pip install httpx requests apscheduler pymupdf4llm pymupdf
pip install numpy scikit-learn
```

### 1.3 前端依赖安装
```powershell
cd frontend
npm install --legacy-peer-deps
```

## 二、配置文件

### 2.1 创建 `.env` 文件
在项目根目录创建 `.env` 文件，内容如下：

```env
# 硅基流动LLM配置
SILICONFLOW_API_KEY=你的API密钥
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1
SILICONFLOW_MODEL=Qwen/Qwen2.5-72B-Instruct

# LLM参数
LLM_MAX_TOKENS=2048
LLM_TEMPERATURE=0.1
LLM_STREAM_TIMEOUT=60

# 嵌入模型配置
ENABLE_TRANSFORMER_EMBEDDINGS=true
ENABLE_TRANSFORMER_RERANK=false

# 服务配置
API_HOST=0.0.0.0
API_PORT=8000
API_DEBUG=false
```

### 2.2 模型文件
确保以下模型已下载到 `models/hub/` 目录：
- `BAAI/bge-large-zh-v1.5`（嵌入模型，1024维）

## 三、启动后端

### 3.1 启动命令
```powershell
# 在项目根目录执行
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

### 3.2 验证后端
浏览器打开 `http://localhost:8000/` ，应返回：
```json
{"name": "OpenEvidence 临床证据助手", "version": "1.0.0", "status": "running"}
```

### 3.3 后端API列表
| 路径 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 服务信息 |
| `/health` | GET | 健康检查 |
| `/api/admin/stats` | GET | 知识库统计 |
| `/api/admin/documents` | GET | 文档列表 |
| `/api/chat/query` | POST | 非流式问答 |
| `/api/chat/stream` | POST | 流式问答(SSE) |
| `/api/eval/run` | POST | 运行评测 |

## 四、启动前端

### 4.1 启动命令
```powershell
cd frontend
npm run dev
```

### 4.2 访问前端
浏览器打开 `http://localhost:5173` （如端口被占用会自动使用5174）

### 4.3 前端页面
- `/chat` - 循证问答（主页面）
- `/admin` - 知识库管理

## 五、数据准备

### 5.1 数据库
- 位置：`data/clinical_kb.db`（SQLite）
- 包含 `table_article_meta` 表，存储文献元数据和全文

### 5.2 向量数据库
- 位置：`data/chroma_db/`
- 使用ChromaDB + 余弦距离
- 嵌入模型：BGE-large-zh-v1.5（1024维）

### 5.3 重建向量库
当切换嵌入模型或新增文献后，需重建向量库：
```powershell
python scripts/rebuild_vector_db.py
```

## 六、常见问题

### Q: 后端启动报 ModuleNotFoundError
```powershell
pip install 缺失的模块名
```

### Q: 前端页面空白
检查 `frontend/vite.config.js` 中代理目标是否指向 `http://localhost:8000`

### Q: LLM问答返回429错误
硅基流动API速率限制，更换模型（如 `Qwen/Qwen2.5-72B-Instruct`）或稍后重试

### Q: 嵌入模型加载失败
确保 `models/hub/` 下有完整的模型权重文件（非0字节）
